WM-811K · ResNet34

Instructions 晶圓缺陷歸類・使用方法與重新訓練指南

本頁為 Wafer Defect Classifier 的獨立說明文件，涵蓋單張圖片與整批 Lot Log 兩種歸類方式、標籤修正流程，以及重新訓練模型的資料策略、硬體建議與訓練時間估算。

晶圓缺陷歸類・使用方法總覽

本工具會判斷一片晶圓圖屬於 8 種缺陷模式（Center / Donut / Edge-Loc / Edge-Ring / Loc / Random / Scratch / Near-full）中的哪一種。頁面上方有 Single Wafer 與 Whole Lot 兩個分頁，可依情況選擇其中一種方式輸入資料；模型推論在本機執行，圖片與 log 資料 不會上傳到外部伺服器。

1Single Wafer：單張圖片歸類

已經有現成的晶圓圖（wafer map）圖檔時使用。

  1. 切到 Single Wafer 分頁，把晶圓圖（PNG / JPG，任意尺寸）拖曳到框內，或點擊選擇檔案。
  2. 按下 「執行辨識」。
  3. 右側「Prediction」區塊會列出 8 種缺陷模式的機率排序，最上方即為模型判定的結果與信心度。

若還有這片 wafer 的 test log CSV，可以在同一分頁下方「Fail Die 座標分佈」區塊 另外上傳，直接畫出該片實際 fail die 的座標分佈圖，與圖片辨識互相獨立、可搭配對照。

2Whole Lot：整批 Lot Log 歸類

手上是整批 lot 的 ATE test log（dlogTDO 格式 CSV）， 不需要自己先轉成圖片。

  1. 切到 Whole Lot 分頁，三種輸入方式擇一：
    • 拖曳多個 CSV 檔案到框內
    • 拖曳整個 lot 資料夾到框內（自動遞迴讀取內部所有 .csv）
    • 點「選擇多個 CSV 檔案」或「選擇 Lot 資料夾」用檔案總管挑選
  2. 確認下方檔案清單無誤後，按 「執行批次辨識」。
  3. 結果會顯示 Lot Summary（Lot ID、wafer 數量、整批 pattern 分佈）與 每片 wafer 一列的明細（wafer ID、良率、Pass/Fail 顆數、判定 pattern、 信心度、小圖預覽），點擊任一列可展開該片完整 8 類機率排序。

支援的 CSV 格式與判定邏輯

程式會自動從 CSV 中的 XAdr / YAdr（座標）與 Bin#（1 = Pass，其他 = Fail）還原出晶圓的 bin map，直接餵給模型判斷， 不需要額外產生圖片檔。支援 CRAFT 系列測試機台輸出的 dlogTDO log：開頭會有 Lot ID、Wafer ID 等欄位，接著是以 Serial#,Site#,Bin#,SBin#,XAdr,YAdr,... 為表頭的資料表。 若上傳的檔案不符合這個格式，該檔案會被略過並在結果下方顯示錯誤原因，不會中斷整批處理。

程式也會自動解析 CSV 表頭裡 IR0(8)、 VF1(17) 這類「測項名稱(Bin#)」的對照表，算出每片 wafer 主要 fail 在哪個測項，顯示在結果表格的「主要 Fail 測項」欄位。

良率門檻（Whole Lot 分頁上方「良率 ≥ 此門檻直接判定 None」）可調整：良率高於門檻的 wafer 會直接判定為 None，不會再跑一次 AI 模型，藉此加快批次處理速度。

修正模型判斷錯誤的標籤

如果模型判斷的 pattern 不準確，可以在結果表格最右邊「修正標籤」欄位手動修正：

  1. 從下拉選單選擇正確的 pattern（或選「不確定/其他」）。
  2. 按 「存」。

儲存後會累積到 corrected_labels/ 資料夾：

corrected_labels/ ├── labels.csv # 每筆修正的紀錄（wafer ID、原判斷、修正後標籤、主要 fail 測項…） └── binmaps/ ├── RK30906-02.npy # 該片 wafer 完整的 Bin# 原始數據（不是圖片，可精確還原） └── ...

labels.csv 可以直接用 Excel 打開檢視。這些修正資料 本身只會被儲存，不會自動拿去重新訓練模型——如果想用它們來 fine-tune， 可以參考下方「重新訓練模型」章節。

重新訓練模型（進階）

當累積了一定數量的「修正標籤」後，可以考慮拿這些資料重新訓練（fine-tune）模型， 讓它學會你標註過的判斷。以下是建議的作法與硬體需求，並提供一個小工具，依你調整的 參數自動產生對應的訓練程式碼。

建議的資料混合策略

  1. 把原始 WM-811K（17 萬筆）與 corrected_labels/ 裡累積的修正資料，混合成同一個訓練集。
  2. 修正資料因為量少，訓練時要給更高的取樣權重（讓它們被多抽到幾次）， 否則會被 17 萬筆原始資料稀釋到幾乎沒有影響，模型學不到你標註的內容。
  3. 保留一份只從原始資料切出來的驗證集，重新訓練後拿新模型跟舊模型在 同一份驗證集上比較，確認沒有變差再替換 best_radai_resnet.pt。
  4. 如果不想處理 17 萬筆這麼大的資料量，可以折衷：每個類別只抽樣一個 代表性子集（例如各 1000～2000 筆）跟修正資料混合訓練，效果通常也足夠，且訓練時間與 資料處理負擔小很多——這是小規模 fine-tune 情境下的常見實務做法。

建議硬體規格

這個模型不大（ResNet34、單通道、64×64 輸入），跟訓練大型語言模型完全不是同一個量級：

產生訓練程式碼

調整下方參數後按「Train Model」，會依你設定的值產生對應的 train.py 程式碼（混合原始資料 + 修正資料、修正資料加權取樣、驗證集只取自原始資料）。 這個按鈕只會在瀏覽器裡產生程式碼文字，不會真的執行訓練——複製或下載後， 還是要在有 Python / PyTorch 環境的電腦上實際執行。

常見問題

      
Q: 我的圖片不是標準晶圓圖格式，可以用嗎？

      

        模型接受任意大小的灰階圖片，內部會自動縮放到 64×64 並正規化。但準確率會依圖片與
        訓練資料（WM-811K 標準晶圓圖）的相似程度而定——一般拍照或非晶圓圖片的預測結果
        不具參考價值。
      

      
Q: 上傳的 CSV 沒有反應 / 顯示錯誤？

      

        請確認檔案是 CRAFT 系列測試機台輸出的 dlogTDO 格式，且包含
        Serial#,Site#,Bin#,SBin#,XAdr,YAdr 表頭。不符合格式的
        檔案會被自動略過，並在結果下方顯示原因，不會中斷整批處理。
      

      
Q: 模型判斷錯了怎麼辦？

      

        在該片 wafer 那一列的「修正標籤」欄位選擇正確 pattern 並按「存」即可，詳見上方
        「修正模型判斷錯誤的標籤」說明。
